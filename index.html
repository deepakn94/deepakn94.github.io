Transactions in a key-value store: This was a final project for MIT's Graduate Distributed Systems class (6.824).
The main goal here is to add the ability to handle atomic, cross-shard transactions into a sharded and replicated key-value store. NoSQL systems are increasingly becoming more popular -- the ability to perform a sequence of multi-record operations atomically makes these systems even more powerful, and in some cases, a viable substitute for a full-fledged database system.
To avoid spurious recomputation associated with protocols such as Two-phase commit and Two-phase locking, our solution made use of a server-side commit protocol called linear transactions. Our implementation was able to guarantee serializability of operations. We used Paxos to ensure all replicas in a replica group perform all operations in the same order.
A more in-depth writeup of our design is available here.

SplitSecure: This was a final project for MIT's Graduate Computer Systems Security class (6.858).
The premise here is that databases of several high profile companies such as Sony, LinkedIn, and Adobe have been compromised in recent years. These security breaches affect millions of users and result in millions of dollars in damages along with a lot of negative press. Our project aimed at mitigating the damage caused by such attacks.
We designed a scalable distributed server-side password storage system with the objective of forcing adversaries to compromise multiple servers in order to obtain any sensitive information -- we made use of a variation of Shamir's secret sharing scheme to ensure that data stored on any one machine in the system is not sufficient to gain access to a user's password.
More details are available here.

ATP Tennis Fantasy engine: This was a final project for MIT's Graduate Machine Learning class (6.867).
We built a Machine Learning engine capable of predicting the winners of a tennis draw as accurately as possible. Our writeup is available here.

Internships
Bing Ads, Microsoft: I implemented a Machine Learning experimentation framework for the Bing Ads Click Prediction team. The tool I built allowed engineers to experiment with different features much more effectively and efficiently.

Data infrastructure team, Pinterest: I helped build a framework that enabled more decisions made at the company to be data-driven. My main project was trying to improve the reliability and efficiency of Pinterests' pipelined data workflows, by moving most workflows off of Amazon's Elastic MapReduce and onto a new platform called Qubole.
I also worked on some other smaller projects over the summer, including an addin that allowed our workflow manager to schedule jobs based on priority; and a verification system that allowed us to hold off on running Hadoop jobs until the job's inputs were ready to be consumed on S3.

Multilingual Systems group (MLS), Microsoft Research India: I designed and implemented a type-ahead email search system in C# for Microsoft Outlook. Autocompletion is a fairly well-solved problem in web search since most people search the web in largely the same way, but autocompletion in email search is a lot harder, since different people find different emails more important. I built a machine learning model that given an incomplete query, attempted to find the most relevant emails in the user's inbox (this would help us rank different candidate query completions).
